  4. Implement the following function in the PyDev module functions.py and test it from a PyDev module named :

    def merge_letters(fh_letter, fh_addresses, fh_merged):
    """
    -------------------------------------------------------
    Merges a letter with a list of addresses.
    Writes the results to a file of completed letters.
    Use: merge_letters(fh_letter, fh_addresses, fh_merged)
    -------------------------------------------------------
    Parameters:
        fh_letter - a file with a letter to process (file - open for reading)
        fh_addresses - a file of addresses (file - open for reading)
        fh_merged - a file of merged letters (file - open for writing)
    Returns:
        None
    -------------------------------------------------------
    """

    This function must call the function get_addresses to extract the addresses.

    The letter to be processed is in letter.txt. Its contents are:

    [forename] [surname]
[street]
[city]

Dear [forename]:

    As a fellow citizen of [city], you and all your neighbours 
on [street] are invited to a celebration this Saturday at 
[city]'s Central Park. Bring beer and food!
--

    To generate the new letter, the various placeholders must be replaced with the matching address data:

    • [forename] with the address forename
    • [surname] with the address surname
    • [street] with the address street
    • [city] with the address city

    Thus, the address:

    ['David', 'Brown', 'King St', 'Waterloo']

    should generate the letter: 

    David Brown
King St
Waterloo

Dear David:

    As a fellow citizen of Waterloo, you and all your neighbours 
on King St are invited to a celebration this Saturday at 
Waterloo's Central Park. Bring beer and food!
--

    Note: this function should assume that the addresses consist of the four fields listed - forename, surname, street, and city - but should work with any letter file in which the placeholders are in a different order, are missing, or appear an arbitrary number of times.

  6. Implement the following function in the PyDev module functions.py and test it from a PyDev module named :

    def substitute(fh_in, fh_out, ciphertext):
    """
    -------------------------------------------------------
    Encipher a file using the letter positions in ciphertext.
    Only letters are enciphered, and the resulting strings are
    in upper case.
    Use: substitute(fh_in, fh_out, ciphertext)
    -------------------------------------------------------
    Parameters:
        fh_in - input file (file - open for reading)
        fh_out - output file (file - open for writing)
        ciphertext - ciphertext alphabet (str)
    Returns:
        None
    -------------------------------------------------------
    """

    This function uses a substitution cipher in which a letter is replaced with another based upon a substitution or ciphertext alphabet. To encipher text, look up the position of a letter in the standard alphabet and replace it with the letter in the same position in the ciphertext alphabet.

    Plaintext alphabet: ABCDEFGHIJKLMNOPQRSTUVWXYZ
    Ciphertext alphabet: DAVIBROWNZCEFGHJKLMPQSTUXY

    In this example A is replaced by D, B is replaced by A, C is replaced by V, and so on. Leave all other characters unchanged. Convert the input to uppercase for the enciphering and make the output entirely uppercase. The function should work with an empty file.

    Use the file pelee.txt.

    Sample testing:

    Input file: pelee.txt
Output file: pelee_substitute.txt
Enter ciphertext: DAVIBROWNZCEFGHJKLMPQSTUXY

Done

    In this example, the first line of pelee.txt:

    It appears that Ontario's Point Pelee National Park has lost its

    should become:

    NP DJJBDLM PWDP HGPDLNH'M JHNGP JBEBB GDPNHGDE JDLC WDM EHMP NPM

    What happens if the ciphertext and the alphabet are the same?
